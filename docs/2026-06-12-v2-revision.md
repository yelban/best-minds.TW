# v2.0.0 修訂記錄：從 expert roleplay 到模擬器思維

> "A good chunk of people misunderstood this tweet btw, which is my bad."
> — Andrej Karpathy

本 skill 的 v1 正是那個「被誤解的版本」之一。本文完整記錄 2026-06-12 的修訂過程：如何從一手資料發現 v1 偏離了方法論本意、如何改造成 plugin、以及用 X 即時搜尋查到的 2026 年後續發展如何變成 v2 的護欄。

## 一、起點：取得兩份一手資料

用 `baoyu-danger-x-to-markdown` 與 `baoyu-youtube-transcript` 抓回方法論的兩個源頭：

1. **模擬器推文 thread**（[x-to-markdown/karpathy/1997731268969304070/](../x-to-markdown/karpathy/1997731268969304070/1.md)，2025-12-07）
   - 第 1 則：「Don't think of LLMs as entities but as simulators. ... There is no "you". Next time try: "What would be a good group of people to explore xyz? What would they say?"」
   - 第 2 則（澄清）：「A good chunk of people misunderstood this tweet btw, which is my bad. I am not suggesting people use the old style prompting techniques of "you are an expert swift programmer" or etc. it's ok.」
2. **《State of GPT》演講字幕**（[youtube-transcript/.../transcript.md](../youtube-transcript/microsoft-developer/state-of-gpt-brk216hfs/transcript.md)，Microsoft Build 2023）
   - 「LLMs don't want to succeed, they want to imitate. You want to succeed, and you should ask for it.」（00:30:16）
   - 「Say something like, you are a leading expert on this topic. Pretend you have IQ 120...」（00:31:33）

> 註：以上兩個目錄是本機工作產物，已列入 `.gitignore`，不隨 repo 發布。

**順手發現的 bug**：v1 的 README 與 docs/origin.md 引用的「原始推文」連結是 `1816531576228053133`——那是 2024 年 7 月的 Jagged Intelligence 推文，不是模擬器推文。引用源頭就錯了。

## 二、反思：v1 的問題

對照兩份資料後浮現一條關鍵演化線——**Karpathy 自己的立場在 2023 → 2025 之間變了**：

| | 2023《State of GPT》 | 2025 模擬器推文 |
|---|---|---|
| 邏輯 | 訓練資料好壞混雜，模型預設模仿全部，用「你是專家」調到高品質區段 | 前沿模型預設輸出已在高品質區段，頭銜無增益（"it's ok"） |
| 性質 | **品質增益技巧** | **認識論工具**：取出多元真實視角 |

v1 SKILL.md 的問題：

1. **原則二「找最強的」與澄清推文直接牴觸**——推文原話是 "a good **group** of people to **explore**"，重點是視角組合，不是單點最強；照 v1 執行容易退化成被撤回的 expert roleplay
2. **完全沒收錄澄清推文**——沒有防誤讀護欄
3. **「引用原話」沒有查證機制**——是幻覺邀請函
4. **與新版 Claude Code 系統提示詞有張力**——harness 要求「lead with the outcome」，多專家長篇輸出需要明定收斂規則
5. **沒有操作流程**——只是一頁心法，執行品質看運氣
6. **在地化問題**——「TA」（中國網路用語）、半形引號

同時辨識出新模型時代（Fable 5、Opus 4.8）的價值轉移：**品質增益已死，反諂媚與定位分歧成為本 skill 最強的存在理由**；且新模型對真實人物公開言論掌握更廣，模擬保真度上升；新工具（WebSearch、Agent subagent、AskUserQuestion）提供 2023 年做不到的執行架構。

## 三、結構改造：plugin 化

仿照 `saygoal.TW` 的單一 plugin repo 模式（`source: "./"`）：

```
best-minds.TW/
├── .claude-plugin/
│   ├── marketplace.json          # Marketplace 清單
│   └── plugin.json               # Plugin 清單
├── skills/best-minds/SKILL.md    # 唯一正本
├── SKILL.md                      # symlink → 正本（舊式安裝相容）
├── docs/
├── README.md / CLAUDE.md / LICENSE
```

三種安裝方式並存：`/plugin marketplace add` + `/plugin install`、`npx skills add`、手動 symlink。

## 四、Grok CLI 查到的關鍵後續發展

用 Grok CLI（X 即時搜尋）三路並行查詢：原 thread 的回應、Karpathy 2026 年後續、「LLM as simulators」概念圈討論。

### Karpathy 本人的 2026 延伸

- **本體論修正**（[原 thread 回覆](https://x.com/karpathy/status/1997759548543947249)）：承認 RLHF 確實造出工程化的「you」（複合人格），但它是後天拼裝（bolt-on）的，不是長期形成的心智；在不可驗證領域尤其難判斷該給多少可信度
- **從面板到母體**（[2026-02-12，投資 Simile](https://x.com/karpathy/status/2022041235188580788)）：「Why simulate one 'person' when you could try to simulate a population? How do you build such a simulator? How do you manage its entropy? How faithful is it?」——模擬器思維的正史方向，entropy 管理與保真度被點名為未解問題
- 另有 [2026-04-10 的 brain upload 討論](https://x.com/karpathy/status/2042626702459674801)：LLM simulators 是「可行版的 brain upload」（與本 skill 較遠，僅記錄）

### 社群實證

- **衝突勝過中性面板**（[Brian Roemmele 自述實驗](https://x.com/BrianRoemmele/status/1998068828295877011)，12,400 條推理 trace、六個前沿模型，**未經同儕審查**）：盲評中「5–7 個立場強烈衝突的人格鏈式辯論」8.72/10，顯著勝過「中性專家面板」6.81 與「無人格」5.94（p < 0.001）。解釋：對齊後模型仍高度獎勵討好，乾淨面板易落入低能量、表面平衡的盆地
- **persona 不提升準確率**（Ethan Mollick 等，[SSRN](https://papers.ssrn.com/sol3/papers.cfm?abstract_id=5879722)）：「你是偉大物理學家」對事實準確率幾乎無顯著提升——佐證「價值不在答得更好」
- **NeurIPS 2025 失效模式**（經 [@koylanai 整理](https://x.com/koylanai/status/1999192104850133146)）：
  1. **假共識** — 多 agent 模擬收斂到訓練資料中位數，少數派被平均掉
  2. **身分扁平化** — 退化成刻板印象；應追求 reasoning fidelity 而非 speech simulation
  3. **合成人格系統性偏移** — 捏造人格有正向漂移、政治傾向漂移；「extract from real，不要 invent」

### 理論限定與倫理

- **janus 澄清**（[2026-04-03](https://x.com/repligate/status/2039857049312612571)）：《Simulators》理論針對 base model；RL 後訓練打破天真模擬器性質，character「常無法化約為訓練資料的線性總和」。意義：我們操作的是「對齊後的模擬器」，預設助理人格是真實的引力井
- **倫理警示**（[@RVAwonk thread](https://x.com/RVAwonk/status/1952851924090233297) 等）：AI 模擬真實人物（尤其已故者）涉及同意權、哀悼操縱、冒充背書，且幾乎不受監管

### 一個有趣的張力

Roemmele 說強衝突人格**有用**（推理深度），Mollick 說 persona **無用**（事實準確率）——兩者其實不矛盾：persona 對封閉事實題無增益，對開放探索題的價值在製造立場張力。這正好對應 skill 流程第 1 步「封閉的事實問題直接回答，不要套面板」。

## 五、套用的修正

### 第一輪（方法論升級 + plugin 化）

1. 收錄澄清推文，新增「這不是 expert roleplay」護欄節，明文禁止「貼專家名字就照常回答」
2. 原則二改寫：「找最強的」→「選探討組合，不是選頭銜」，面板至少放一位唱反調的
3. 新增五步流程：判斷問題性質 → 選人 → 查證 → 分聲部模擬 → 收斂綜合
4. 引言查證規則：查證或明標轉述，不可捏造名言
5. 平行 subagent 面板（進階）
6. 在地化修正：移除「TA」、半形引號改「」；修正錯誤的源頭推文連結

### 第二輪（吸收 2026 後續發展）

1. **進階面板改兩階段**：獨立陳述輪（防錨定）→ 交鋒輪（吸收 Roemmele 發現，逼出深度）→ 收斂
2. **新增「已知失效模式」節**：假共識（禁止「各有道理」收尾，分歧寫到「誰會對誰的哪個論點說不」）、身分扁平化（reasoning fidelity 而非模仿腔調）、正向漂移（每個聲部要問「這個人會指出哪些困難」）
3. **原則三強化為「extract 不 invent」**：優先具名真人；捏造複合人格只在找不到真人時用且須註明（範例中的「一位資料庫系統老兵」改為 Michael Stonebraker）
4. **新增原則五「倫理界線」**：只模擬公眾人物公開思想立場、全程標明模擬、不得冒充背書、近期過世者從嚴
5. **本體論軟化**：「沒有那個你」→ Karpathy 修正版「工程化的 bolt-on 複合人格，不是長期形成的心智」
6. **docs/origin.md 補 2026 演化線**：Simile、janus 澄清、Mollick、Roemmele、NeurIPS、倫理警示，逐一標注對應的 skill 護欄

核心領悟一句話：**skill 的價值主張（反諂媚、定位分歧）是對的，但天真執行會自我打臉——面板假共識正是另一種形式的諂媚。**

## 六、留給未來的問題

- **entropy 管理與保真度**：Karpathy 點名的未解問題；面板的視角多樣性如何量化、如何避免坍縮
- **Roemmele 結果待驗證**：自述實驗未經同儕審查；交鋒輪的實際增益值得在使用中觀察
- **population simulation**：從「面板」到「母體」的下一步——若 Simile 式工具成熟，本 skill 的定位可能從「手工選人」演化為「母體取樣」
- **對齊後模擬器的極限**：janus 指出 RL 後模型不是天真模擬器；指定視角能脫離助理人格引力井到什麼程度，沒有定論
