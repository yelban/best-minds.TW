# Best Minds Skill 使用指南

## 能力說明

這是一個**思維方法論** skill，改變 AI 回答開放式問題的方式：

| 傳統問法 | Best Minds 問法 |
|---------|----------------|
| 「你怎麼看？」 | 「世界上哪群人最適合探討這個？他們會怎麼說？」 |
| 預設助理人格的共識答案 | 多位真實人物的視角碰撞 + 收斂綜合 |

適用於開放式判斷、設計取捨、策略方向等沒有單一正解的問題；封閉的事實問題不適用（直接回答即可）。

## 觸發詞

在對話中使用以下詞彙即可觸發：

- 最強大腦
- 頂級專家
- 世界級
- 誰最懂這個
- best minds / who knows this best / panel of experts

## 使用範例

```
「LLM agent 的記憶系統該怎麼設計？誰最懂這個？」
→ 組一個視角圓桌（如 Andrej Karpathy、Richard Sutton、Michael Stonebraker），
  分聲部模擬各自的主張與分歧，最後收斂成建議

「我想辭職創業，請用頂級專家的視角給建議」
→ 可能模擬 Paul Graham，並刻意放一位會唱反調的風險視角（如 Nassim Taleb），
  而不是一面倒的鼓勵
```

人選一律用有公開言論記錄的真實人物；找不到合適真人時才用虛構原型，且會註明。

## 運作流程

圓桌不是黑箱，是一套可驗證的步驟：

1. **測繪對立軸** — 先看這問題有哪些立場光譜，避免選到共享同一盲點的人
2. **選人亮牌** — 列出人選名單：全名 + 一句身分簡介 + 入選理由（如「Charles Packer——MemGPT 論文第一作者、Letta 創辦人」），並明確標出**指定反方**——誰負責唱反調、針對你問題的哪個前提；組合模糊時會先用選項詢問你
3. **檢索接地** — 爭議／時效性強的主題，先查當事人實際近期立場再模擬（查不到就明說是推測，不虛構）
4. **分聲部模擬** — 每個聲部以全名標明，呈現其獨有思考框架與具體主張（不是模仿說話腔調），全程明示是模擬
5. **收斂綜合（含盲區掃描）** — 結尾標出共識、關鍵分歧、以及對你問題的綜合建議；不會用「各有道理」式的表面平衡收尾，分歧無法調和時會明說在什麼前提下該聽誰的。**必含盲區掃描**：這群人共同沒談到什麼、暴露了哪個全體預設。綜合者自己延伸的具體建議（工具選型、數字門檻）會標明為綜合者所加，不掛在人物名下

三人以上可用平行 subagent 跑兩階段（獨立陳述 → 交鋒 → 收斂），各聲部彼此看不到對方，避免互相錨定。

## 回覆語言

`SKILL.md` 正本以英文撰寫（國際推廣與跨語言中立），但開頭明令「以使用者提問的語言回覆」。所以你用繁中問，得到繁中回答；英文使用者問英文，得到英文回答——已於中立環境驗證涵蓋中／英／日三語。詳見 `2026-06-20-language-ab-and-english-canonical.md`。

## 核心價值

利用 LLM 作為「模擬器」的特性——訓練資料包含大量真實人物的著作、演講、訪談，所以能合理模擬這些人的思維框架。價值在於視角多元、反諂媚（模擬人物批評起來毫無顧忌）、定位專家分歧、挖掘集體盲區，而不是給答案掛專家頭銜。

> "Don't think of LLMs as entities but as simulators."
> — Andrej Karpathy
